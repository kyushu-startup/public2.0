生成AIによって発信量は増えた。しかし、経営・現場・市場の前提が揃わなければ、組織は動かない。広報5.5は、マルチAIと人間の判断で一次情報を構造化し、意味ある実務アセットへ変換するための研究仮説です。

This is a working archive for observing semantic drift and designing AI-assisted communication operations.

01 / Observation

発信量は増えた。
前提は、揃わない。

semantic drift

同じ言葉なのに、意味が違う。

「顧客ファースト」「価値提供」「らしさ」が、営業・開発・人事で別々の行動を指している。

inactive philosophy

理念は読まれる。でも、動きは変わらない。

文章としては理解されても、明日から何を変えるべきかが現場に接続されていない。

external / internal gap

社外の期待と、社内の実感が離れる。

採用、営業、広報の言葉が、それぞれ微妙に違う会社像をつくってしまう。

role limitation

広報が、“整える人”で止まる。

発信の前に必要な、認識合わせ・構造化・翻訳までが、広報の仕事として扱われにくい。

02 / Hypothesis

広報は、外へ届ける仕事から、意味を同期する仕事へ。

広報5.5の役割は、きれいな言葉を作ることではありません。経営の意図、現場の実感、市場の期待を行き来しながら、組織が同じ前提で動ける状態をつくることです。

05 / Workflow

情報を、
意味ある構造へ。

Input Layer

経営会議、顧客の声、Slack、採用面談。

散らばった断片を、意味の差分として扱う。

Structure Layer

思考整理、文脈編集、暗黙知の翻訳。

AIは文章生成ではなく、同期の補助線として使う。

Output Layer

採用記事、LP、営業資料、FAQ。

アウトプットは目的ではなく、認識整合の副産物。
